🐛 Bugs Reales Encontrados (12)

#1 — Filtro por moneda USD incluye el curso gratuito

Prueba #8. Al filtrar por USD aparece "Marketing Digital Avanzado" (USD) Y "Introducción a la Programación Web" (gratuito, con currency=USD). El curso gratuito tiene currency=USD en la BD porque es su moneda por defecto, pero pricingModel=FREE. El filtro de moneda debe excluir los cursos gratuitos o el frontend debe aclarar que es gratuito.

Solución: El API debe excluir cursos FREE cuando se filtra por moneda, o devolverlos con una marca especial. → Corregido en API.

#2 — Sesiones bloqueadas en curso gratuito

Prueba #13. En "Introducción a la Programación Web" (gratuito), las sesiones aparecen como "Preview gratuito" pero no son accesibles sin matrícula. El botón de matricularse no aparece.

Causa: El curso gratuito requiere matrícula explícita. Si el usuario no está logueado, no ve el botón de matricularse. Si está logueado pero no matriculado, debería ver "Matricularse". → Revisar flow de matrícula gratuita.

#3 — Error "sin evaluación" no se maneja en frontend

Prueba #39. Al llegar al 100% de progreso en un curso sin evaluación configurada, aparece "Elegible para certificado" con botón "Realizar evaluación". Al hacer click, la API devuelve error "Este curso no tiene evaluación configurada" y se muestra en pantalla como error crudo. → Corregido: ocultar botón si no hay evaluación.

#4 — PDF de certificado descarga como JSON

Prueba #50. Al hacer click en "Descargar PDF" desde el navegador, se muestra JSON en lugar de descargar el archivo. La API devuelve application/pdf correctamente, pero el frontend hace fetch y muestra la respuesta. → Corregido: usar window.open o link directo.

#5 — Instructor no puede crear cursos desde UI

Pruebas #53-60. El instructor inicia sesión pero no tiene panel de gestión de cursos. Los endpoints existen (POST /api/courses, POST /api/courses/{id}/sessions) pero solo vía API, no hay interfaz visual.

Estado: Endpoints funcionales verificados anteriormente. Falta panel de instructor en frontend. → Feature pendiente.

#6 — Panel admin no carga correctamente

Prueba #66. /es/dashboard/admin no muestra el panel de pagos. Posible error de carga del cliente.

Verificar: El panel se renderiza del lado cliente. Requiere sesión ADMIN activa. → Revisar.

#7 — Filtro PAID no visible

Prueba #10. El usuario no ve la opción "De pago" en el selector de tipo de curso. El select tiene las opciones "Todos", "Gratuitos", "De pago". → Verificar renderizado del select.

#8 — Sesiones sin funcionalidad

Prueba #13. Las sesiones preview muestran "Preview gratuito" pero al hacer click no ocurre nada. Los videos son placeholders (YouTube Rick Roll).

#9 — Verificación de badge revocado no comprobable

Prueba #75. No se pudo probar porque el certificado fue revocado en pruebas anteriores y no se generó uno nuevo. → Generar nuevo certificado para prueba.

#10 — Admin features no accesibles desde UI

Pruebas #70-80. Funcionalidades de admin (verificar pagos, revocar certificados, instrucciones, auditoría) funcionan vía API pero el panel admin no carga. → Mismo root cause que #6.

#11 — Descripción de fallos confusa por números desfasados

Prueba #21-22, #53-54. La numeración del checklist no coincide porque algunas filas quedaron vacías. El usuario reporta "Login como estudiante → redirige al dashboard" como #21 pero en la tabla es #20.

#12 — No se muestra instructor en sidebar del curso

Prueba #12. El nombre del instructor no aparece en la página de detalle del curso. Se implementó el endpoint pero el frontend no lo renderiza. → Verificar integración.
