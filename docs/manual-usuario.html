2. Visitante Sin autenticación

2.1 Ver Catálogo de Cursos

Prerrequisitos: Ninguno.

Pasos:

  1. Acceder a /es/courses
  2. Usar los filtros de moneda (CUP/USD/EUR) y tipo (Gratuito/De pago)
  3. Usar la barra de búsqueda para encontrar cursos por nombre

2.2 Registrarse

Prerrequisitos: No tener una sesión activa. Email no registrado previamente.

  1. Acceder a /es/register
  2. Completar: nombre, email y contraseña (mínimo 6 caracteres)
  3. Click en "Registrarme"
  4. Redirige automáticamente al Dashboard

2.3 Iniciar Sesión

Prerrequisitos: Tener una cuenta registrada. No estar bloqueado por rate-limiting (3 intentos fallidos = bloqueo temporal).

  1. Acceder a /es/login
  2. Ingresar email y contraseña
  3. Click en "Entrar"

2.4 Verificar un Certificado

Prerrequisitos: Tener el Badge ID de un certificado emitido.

  1. Acceder a /es/verify/{badgeId}
  2. La página muestra: estado (válido/revocado), estudiante, curso, fecha de emisión

3. Estudiante STUDENT

3.1 Dashboard del Estudiante

Prerrequisitos: Haber iniciado sesión.

El dashboard muestra dos secciones: Mis Cursos (cursos matriculados con barra de progreso) y Mis Certificados (certificados emitidos).

3.2 Matricularse en un Curso Gratuito

Prerrequisitos: Haber iniciado sesión. No estar ya matriculado en ese curso.

  1. Ir al catálogo y seleccionar un curso marcado como "Gratuito"
  2. Click en "Matricularse"
  3. La matrícula se crea instantáneamente con estado ACTIVE
  4. Redirige al detalle del curso para comenzar

3.3 Matricularse en un Curso de Pago

Prerrequisitos: Haber iniciado sesión. No estar ya matriculado en ese curso.

  1. Seleccionar un curso de pago y click en "Matricularse"
  2. En el checkout, Paso 1: Seleccionar método de pago (EnZona, Transfermóvil, Crypto USDT, etc.)
  3. Paso 2: Leer las instrucciones de pago y datos de cuenta
  4. Paso 3: Realizar el pago y subir el comprobante (imagen o PDF)
  5. Click en "Enviar" — el pago queda en estado PENDIENTE
  6. Un administrador revisará el comprobante y activará el acceso
⏳ Hasta que el admin no verifique el pago, el curso aparecerá como "Pendiente de aprobación" y no se podrá acceder a las sesiones.

3.4 Completar Sesiones

Prerrequisitos: Estar matriculado con estado ACTIVE.

  1. Abrir el curso desde el dashboard o catálogo
  2. Ver la lista de sesiones: las completadas muestran ✅, las pendientes tienen botón "Completar"
  3. Click en "Completar" para cada sesión
  4. La barra de progreso se actualiza automáticamente
⚠️ Las sesiones marcadas como "Vista previa" son accesibles sin matrícula. Las demás requieren matrícula activa.

3.5 Realizar Evaluación

Prerrequisitos: Haber completado el 100% de las sesiones del curso. Que el curso tenga una evaluación configurada.

  1. Al llegar al 100% de progreso, aparece la sección "Elegible para certificado"
  2. Click en "Realizar evaluación"
  3. Responder las preguntas (opción múltiple, verdadero/falso, respuesta corta)
  4. Enviar — la corrección es automática
  5. Se necesita ≥80% para aprobar. Si se reprueba, se puede reintentar.

3.6 Emitir Certificado

Prerrequisitos: Haber aprobado la evaluación (≥80%). No tener ya un certificado emitido para este curso.

  1. Tras aprobar, click en "Emitir certificado"
  2. El sistema genera un Open Badge 3.0 con firma criptográfica EdDSA
  3. Opciones: Descargar PDF, Verificar online, compartir Badge ID
  4. El certificado aparece en "Mis Certificados" del dashboard

3.7 Cerrar Sesión

Prerrequisitos: Tener una sesión activa.

Click en "Cerrar Sesión" en la navbar. Las cookies se eliminan y se redirige al inicio.

4. Instructor INSTRUCTOR

4.1 Crear un Curso

Prerrequisitos: Haber iniciado sesión como INSTRUCTOR.

Vía API:

POST /api/courses
Body: {"slug":"mi-curso","title":{"es":"Título","en":"Title"},"description":{"es":"Desc","en":"Desc"},"pricingModel":"FREE","currency":"USD"}

El slug debe ser único. El instructor queda asignado automáticamente como creador del curso.

4.2 Añadir Sesiones a un Curso

Prerrequisitos: Ser el instructor del curso (o ADMIN). El curso debe existir.

POST /api/courses/{id}/sessions
Body: {"title":{"es":"Sesión 1","en":"Session 1"},"description":{"es":"...","en":"..."},"sessionType":"RECORDED","preview":true,"videoUrl":"...","order":1}

Tipos de sesión: RECORDED (grabada), LIVE (en vivo), HYBRID (híbrida).

Si preview: true, la sesión es visible sin matrícula.

4.3 Editar un Curso

Prerrequisitos: Ser el instructor del curso (o ADMIN).

Mismo endpoint POST /api/courses incluyendo el id del curso.

5. Administrador ADMIN

5.1 Gestionar Pagos Pendientes

Prerrequisitos: Haber iniciado sesión como ADMIN. Que existan pagos en estado PENDING.

  1. Acceder a /es/dashboard/admin
  2. Ver lista de pagos pendientes con: estudiante, curso, método, monto, comprobante
  3. Verificar: Click en "Verificar Pago" → el pago pasa a VERIFIED y la matrícula se activa
  4. Rechazar: Escribir motivo y click en "Rechazar Pago" → el pago pasa a REJECTED

5.2 Gestionar Certificados

Prerrequisitos: Haber iniciado sesión como ADMIN. Que existan certificados emitidos.

GET /api/admin/certificates — Listar todos
POST /api/admin/certificates/{id}/revoke — Revocar con motivo

Un certificado revocado se muestra como "❌ Revocado" en la verificación pública.

5.3 Gestionar Instrucciones de Pago

Prerrequisitos: Haber iniciado sesión como ADMIN.

GET /api/admin/payment-instructions — Listar
POST /api/admin/payment-instructions — Crear/editar (con id para editar)

Se puede activar/desactivar (isActive: false) y restringir por país (geoRestriction: "CU").

5.4 Auditoría

Prerrequisitos: Haber iniciado sesión como ADMIN.

GET /api/admin/audit-log — Consultar registro

Filtros opcionales: ?entity=Payment&action=PAYMENT_VERIFIED
